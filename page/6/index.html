js特效-事件冒泡

什么叫冒泡?

触动一个元素的事件的时候,该元素的所有祖先元素的该事件都会被执行,这就叫做冒泡;

如果祖先元素没有绑定该事件,那么也会发生冒泡;因为该元素还是有这个事件的只不过,

事件的执行程序是null;

事件的传播有的时候我们主动利用,但是更多的时候我们在阻止传播;

1.火狐谷歌ie9+: 阻止冒泡和阻止捕获是一个方法;
2.ie678      : 可以阻止冒泡,但是无法阻止捕获因为ie678没有捕获;

js-特效-事件解绑

事件解绑就是把已经绑定的事件解除; (怎么绑定就怎么解绑)

1.直接绑定;

   document.onclick = fn1;
直接解绑:  null;
   document.onclick = null;

2.addEventListener事件解绑；(对火狐谷歌ie9+)

   document.addEventListener("click",fn2);
removeEventListener();
   document.removeEventListener("click",fn2);

3.attachEvent事件解绑；(对ie678)

    document.attachEvent("onclick",fn3);
解绑：detachEvent();
    document.detachEvent("onclick",fn3);
  • 注意点：
    1.怎么绑定，怎么解绑；
    2.只能一个一个解绑；不能一次解绑一群；
    3.必须传递满两个参数；
    4.匿名函数无法解绑；
    拓展：jquery会对事件的操作进行优化；

js-特效-事件监听

为什么会出现事件监听?

因为普通事件绑定容易出现层叠;

事件监听

你做你的,我做我的,但是我监听着你!程序中,两个逻辑同一事件都执行;

  • addEventListener(); 火狐谷歌ie9+
    1.参数有三个;
    (1).事件名;
    (2).逻辑;(匿名函数,函数名)
    (3).boolean;支持冒泡还是捕获;
    2.先绑定谁,执行谁;事件监听不会自己和自己矛盾;
    3.ie678不支持;

    ie678: attachEvent();
    1.参数有两个;
    (1).事件名(带on);
    (2).逻辑;(匿名函数,函数名)
    (3).没有第三个参数:ie678只支持冒泡不支持捕获;
    2.先绑定谁,后执行;事件监听不会自己和自己矛盾;(执行先后影响不大)

js-特效-offset系列

offset检测盒子宽高的时候包括盒子的宽高本身和padding/border;不包括margin;

  如果要用offset检测盒子宽高的时候,最好知道盒子本身不包括padding和border在使用;
offsetWidth/offsetHeight = width/height + padding + border;

offsetWidth 和 offsetHeight: 用于检测盒子的宽高;

用法:
    DOM元素.offsetWidth   DOM元素的宽
    DOM元素.offsetHeight  DOM元素的高

offsetTop和offsetLeft: 用于检测盒子距离父系盒子中带有定位的盒子的距离,如果都没有模拟距离页面的距离;

1.父系盒子有定位,距离最近的带有定位的父盒子的距离;
2.父系盒子没有定位,距离页面距离;和本身有没有定位没有关系;
用法:
    DOM元素.offsetWidth   DOM元素的宽
    DOM元素.offsetHeight  DOM元素的高

offsetTop/Left总结：

1.获取盒子在页面中的坐标；（父系盒子都没有定位；一般都是获取元素在页面的位置）
2.获取盒子的top值和left值；（子绝父相，获取子盒子的top和left值）

offsetParent: 用于检测盒子父系盒子中带有定位的盒子(父元素);

1.父系盒子有定位,获取最近的带有定位的父盒子;（找元素）
2.父系盒子没有定位,获取body,和本身有没有定位没有关系;

offsetTop 和 style.Top的区别

一、最大区别在于offsetTop可以返回没有定位盒子的距离上侧的位置。而 style.Top不可以

二、offsetTop 返回的是数字，而 style.top 返回的是字符串，除了数字外还带有单位：px。
三、offsetTop 只读，而 style.top 可读写。（只读是获取值，可写是赋值）
四、如果没有给 HTML 元素指定过 top 样式，则style.top 返回的是空字符串。

总结:

1.style.left: 一般用于赋值;(他可以获取值但是只能获取行内式;)
2.offsetLeft: 一般用于获取值;(因为获取值不带单位,而且不仅行内式,外链内嵌都能获取;)

js-BOM

简介:BOM提供了独立于内容而与浏览器窗口进行交互的对象

由于BOM主要用于管理窗口与窗口之间的通讯，因此其核心对象是window
BOM由一系列相关的对象构成，并且每个对象都提供了很多方法与属性
BOM缺乏标准，JavaScript语法的标准化组织是ECMA，DOM的标准化组织是W3C
BOM最初是Netscape浏览器标准的一部分

BOM: Browser Object Model; 把整个浏览器做为一个对象,抽象成一个模型;(由属性和方法构成)

属性和方法: 自定义的和默认就有的;

open(); 就是打开一个新窗口;

1. 第一种: 只有一个参数;  打开一个新页面,参数作为地址;
window.open("http://www.baidu.com")
2. 第二种: 有两个参数;    页面跳转(当前页面或者新页面);
window.open("http://www.baidu.com","_self"/"_blank")
2. 第三种: 有三个参数;    可以设置新窗口的参数;
window.open("http://www.jd.com","_blank","width=200,height=200,top=100,left=100");

注:window 作为顶级全局对象,可以省略

close(); 就是关闭一个窗口;只有一个参数;

window.open("http://www.baidu.com")

  • window – BOM核心
    其下面有很多属性，在这么多的属性中只要讲三个：这三个属性的值都是对象；

    1. location 对象 地址的意思； （属性和方法）

    2. navigator: 和浏览器和系统有关； （属性）

    3. history: 和前进后退刷新； （方法）
      History对象，其实就是来保存浏览器历史记录信息。
      history.back(); 后退;
      history.forward(); 前进
      history,go(); history.go(1);//-1后退; 0刷新; 1前进
